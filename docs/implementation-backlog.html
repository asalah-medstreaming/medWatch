Task information is sourced from all backlog HTML pages in the documentation portal. Sprint assignments and dependencies reflect the latest state of each backlog page.

Total Tasks

Implementation Sprints

Business Value

Effort Profile

Foundation and core layers drive the largest setup effort and unblock all later sprints.

2. Dependency Flow

Foundation Layer → Core Platform Layer → Operational Layer → Automation Layer → Presentation Layer → Advanced Features

  • Sprint 1Foundation: access control, asset enrollment, data intake, dashboard, infrastructure monitoring
  • Sprint 2Core Platform: availability, data exchange, queue, service health monitoring
  • Sprint 3Operational: alerting, incident management, log investigation, notifications
  • Sprint 4Automation: automation management, maintenance windows, remediation, rule engine
  • Sprint 5Presentation: business process monitoring, reports
  • Sprint 6Advanced: AI management, auto-update management
